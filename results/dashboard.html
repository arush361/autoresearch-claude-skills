What Was Improved in the Skill

A single, high-impact mutation was added to ProductNew.md that transformed the skill from 59.2% to 97.5% pass rate across 20 runs and 6 eval criteria.

The Mutation: "Projected Totals (MANDATORY)"

A new section was added after the WINNING Filter Scoring block that requires every output type to include projected WINNING totals and preliminary recommendations, even before the user provides their scores.

Key rules added: 1. Calculate projected WINNING totals /60 using conservative placeholder scores of 6/10 for each user-scored criterion (clearly labeled as "illustrative, assuming average user scores of 6/10"). Show the projected total for every gap/feature. 2. Provide a preliminary FILE/WAIT/SKIP recommendation based on the projected total. Clearly state this is preliminary and will be finalized after user provides their scores. 3. Include a Decision Summary table at the end of every gap analysis, PRD, or backlog output with projected totals and preliminary recommendations. Applied to ALL output types: - Gap analysis (/pm:gaps): Project totals per gap, preliminary FILE/WAIT/SKIP - PRD (/pm:prd): Project the total, state preliminary recommendation - Backlog (/pm:backlog): Project totals per item, assign Now/Next/Later - Landscape (/pm:landscape): Add "Preliminary Gap Signals" section with WINNING projections for 3-5 obvious gaps from research - Product analysis (/pm:analyze): Add "WINNING Preview" section scoring 3-5 likely gaps with projected totals

Why This Worked

The original skill had a structural gap: it correctly implemented hybrid scoring (Claude scores Pain/Timing, user scores 4 others) but never told Claude what to do before the user responds. This left outputs without totals, without recommendations, and without scoring sections for non-gap tasks. The single "MANDATORY" instruction closed all three gaps at once.

Before vs After

Before (Baseline)

  • Landscape output had zero WINNING scoring

After (ProductNew)

  • All gaps show projected total /60 with 6/10 placeholders

  • User sees prioritization signal instantly, refines with their scores
